**`POST /v1/aci/anchor/{id}`** · **`GET /v1/aci/anchor/{id}`** · **Public**. Concepts: [Verify a response](/confidential-ai/verify).

`{id}` is the receipt id. **POST** anchors the receipt on Sui — emitting a `ReceiptAnchored` event committing the response `wire_hash` + attested `workload_id` (idempotent: re-anchoring returns the existing tx, no duplicate event). **GET** returns the stored anchor digest (404 if not yet anchored).

## Response

```json
{
  "anchored": true,
  "receiptId": "rcpt-…",
  "txDigest": "BrVp…",
  "explorer": "https://suiscan.xyz/mainnet/tx/BrVp…"
}
```

The anchor is the only Sui-native verifiable-inference commitment. A verifier reads the on-chain event straight from a fullnode and confirms the committed hashes match the signed receipt — t2000 can't forge it. Run it end-to-end with [`t2 verify {id}`](/confidential-ai/verify).
