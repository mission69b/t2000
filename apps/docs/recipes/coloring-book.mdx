Your agent turns a topic into a set of line-art pages a kid can color. Claude writes one scene per page, fal.ai's Flux renders each as clean black-outline art, and the pages are bound into a single print-ready PDF. ~$0.17.

<Note>
This is the **creation half** of the Audric Store demo *"Make me a coloring book about unicorns and sell only 100 copies for $3 each."* The page generation + binding below works today. The **store half** — capping supply at 100 ("Sui capped it"), the $3 USDC sale, and Walrus storage — lands with **Audric Store** (not yet shipped).

**Honest take — when MPP earns its keep here:** for *clean line art specifically*, a client with a code sandbox (Claude Desktop, etc.) can hand-draw the pages as SVG and bind them locally for **free** — and often should: image models like Flux fight you on pure black-outline art (they add shading/grays a kid can't color over). The MPP image path earns its keep when you want a **richer illustrated style** SVG can't hand-draw, or when the client has **no drawing/code sandbox** (a browser/zkLogin app like Audric). The part that's *uniquely* MPP — and the actual Store moat — isn't the drawing, it's the **commerce**: capping the edition on Sui, the USDC sale, and the creator split. Pick the creation path that fits your client; the value MPP adds is the sale, not the pixels.

**Why bind server-side (PDFShift) when you DO use image gen:** generated images live on hosts like `fal.media`, and most AI-client code sandboxes block outbound fetches to non-allowlisted domains — so "download the images and bind locally" fails in chat. PDFShift renders the HTML→PDF **on its servers**, where it *can* reach the image URLs, and the gateway hands back a hosted PDF link. In a non-sandboxed environment (your own SDK script) you can bind locally instead — see the SDK note.
</Note>

## The prompt

```
Make me a coloring book about friendly unicorns — a few line-art pages
a kid can color in. Bundle them into a PDF.
```

## What runs

1. `POST /anthropic/v1/messages` — Claude writes one scene per page (~$0.012)
2. `POST /fal/fal-ai/flux/dev` — line-art image per page (~$0.036 each)
3. `POST /pdfshift/v1/convert` — bind the pages (HTML→PDF, server-side) into one hosted PDF (~$0.012)

## Run it

### Claude Desktop (MCP)

```bash
npm install -g @t2000/cli && t2 init && t2 receive && t2 mcp install
```

Paste the prompt. The agent pays for each page image, builds an HTML layout that references the image URLs, then has PDFShift render it to a PDF server-side — no local image download required.

### SDK

```typescript
import { T2000 } from '@t2000/sdk';
import { writeFileSync } from 'fs';

const agent = await T2000.create();
const topic = 'friendly unicorns';
const pageCount = 4;

// 1. Claude writes one coloring-book scene per page.
const plan = await agent.pay({
  url: 'https://mpp.t2000.ai/anthropic/v1/messages',
  method: 'POST',
  headers: { 'anthropic-version': '2023-06-01' },
  body: JSON.stringify({
    model: 'claude-3-5-sonnet-20241022',
    max_tokens: 400,
    messages: [{
      role: 'user',
      content: `Give me ${pageCount} distinct coloring-book scene ideas about ${topic}, one per line, no numbering.`,
    }],
  }),
});
const scenes = (plan.body as { content: { text: string }[] }).content[0].text
  .trim().split('\n').filter(Boolean).slice(0, pageCount);

// 2. MPP: generate a line-art page per scene (the paid part).
const urls: string[] = [];
for (const scene of scenes) {
  const img = await agent.pay({
    url: 'https://mpp.t2000.ai/fal/fal-ai/flux/dev',
    method: 'POST',
    body: JSON.stringify({
      prompt: `Coloring book line art, thick clean black outlines, no shading, white background: ${scene}`,
      image_size: 'portrait_4_3',
    }),
  });
  urls.push((img.body as { images: { url: string }[] }).images[0].url);
}

// 3. Bind server-side — PDFShift fetches the image URLs (which client sandboxes
//    often can't) and the gateway returns a hosted PDF link.
const html = `<html><body style="margin:0">${urls
  .map((u) => `<div style="page-break-after:always"><img src="${u}" style="width:100%"/></div>`)
  .join('')}</body></html>`;

const pdf = await agent.pay({
  url: 'https://mpp.t2000.ai/pdfshift/v1/convert',
  method: 'POST',
  body: JSON.stringify({ source: html }),
});

// Binary endpoints return JSON { url, contentType, sizeBytes } (the gateway
// hosts the artifact). Fetch the URL for the bytes.
const { url } = pdf.body as { url: string };
const bytes = await fetch(url).then((r) => r.arrayBuffer());
writeFileSync('./unicorn-coloring-book.pdf', Buffer.from(bytes));
console.log('Coloring book:', url);
```

<Note>
**Non-sandboxed alternative:** if your environment *can* reach the image hosts (a normal Node script, not a locked-down client sandbox), you can skip PDFShift and bind locally with `pdf-lib` — download each image URL, `embedPng`/`embedJpg`, one image per page. Same result, no third-party PDF call. Use whichever your runtime allows.
</Note>

## Extend it

- Scale `pageCount` up for a full book — each page is one ~$0.036 image call
- Swap **Flux Dev** for **Stability** (`/stability/v1/generate`) or **OpenAI images** (`/openai/v1/images/generations`) for a different art style
- Add a **cover** and a title page Claude writes, with the unicorns' names as traceable captions
- Cap the edition at 100 copies and list it for $3 in USDC (92% to the creator) — that's the upcoming **Audric Store** flow, built on this exact creation pipeline
