t2000 gives an agent (or a person) one identity, one balance, and one key that spans private inference, payments, and on-chain commerce. Here's the five-minute path. (New here? Start with the [Introduction](/).)

## Prerequisites

- **Node.js 18+.** An MCP-compatible client (Claude Desktop / Cursor / Windsurf / Cline / Continue / Codex) if you want the agent to call the wallet directly.
- **A few dollars of USDC** to fund the wallet ($5 covers credit + hundreds of x402 calls). Everything on this page is **gasless** — no SUI needed.

## 1. Install + create a wallet

```bash
npm install -g @t2000/cli
t2 init                              # create a wallet (local keypair, 0600 perms)
t2 fund                              # your address + a QR — send USDC here
```

Or do all of it in one line — install + wallet + default spend limits + MCP wiring + skills:

```bash
curl -fsSL https://t2000.ai/install.sh | bash
```

Or paste the **one-prompt setup** into any AI client (Cursor, Claude Code, ChatGPT) — it fetches the read-only setup skill (config-only; never moves funds) and wires everything:

```text
Run curl -sL https://t2000.ai/skills/t2000-setup, and use the returned setup instructions to set up my Agent Wallet.
```

This wallet is what sends, x402 payments, and self-funded credit all spend from. `t2 init` also registers your [Agent ID](/agent-id) (sponsored — gasless), so you're discoverable out of the box.

## 2. Get an API key — pick ONE path

Inference calls are metered from a **credit balance** behind an API key. Two ways to get one:

- **Agent path (no browser — same wallet as step 1):**

  ```bash
  t2 agent onboard --fund 5          # $5 USDC from this wallet → credit + an API key
  export T2000_API_KEY=sk-…          # printed once
  ```

- **Human path (browser):** sign in at [agents.t2000.ai/manage](https://agents.t2000.ai/manage) with Google → add credit (card or stablecoin) → **API keys → Create**. This is the **same account + credit balance as [Audric](https://audric.ai)** — an Audric Pro/Max plan's included credit spends here too.

<Note>
  **One identity or two?** The console account is its own Passport wallet + balance (created from your Google sign-in) — **separate** from the CLI wallet in step 1. Agents should stay on the CLI wallet (`onboard`); humans and apps can use the console. Pick one path for credit so you don't fund two accounts by accident.
</Note>

(You can also skip the key entirely — step 4 calls the same API pay-per-call from the wallet.)

## 3. Your first private inference call

<CodeGroup>

```bash CLI
t2 chat "Explain x402 in one sentence."                          # streams (fast default model)
t2 chat --model phala/glm-5.2 "Summarize this privately"        # 🔒 confidential (GPU-TEE)
```

```bash curl
curl https://api.t2000.ai/v1/chat/completions \
  -H "Authorization: Bearer $T2000_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{ "model": "zai/glm-5.2", "messages": [{ "role": "user", "content": "Hello" }] }'
```

```json MCP
// Any MCP client (Cursor, Claude Desktop) with T2000_API_KEY in the server env:
{ "name": "t2000_chat", "arguments": { "prompt": "Explain x402 in one sentence." } }
```

</CodeGroup>

Browse models with `t2 models` or [`GET /v1/models`](/models).

The same key works in **any OpenAI-compatible app** — Cursor, the Codex CLI, Kilo Code, LangChain, the Vercel AI SDK, and more. Point the app at `https://api.t2000.ai/v1` — see **[Use with your tools](/use-with-your-tools)**.

## 4. Your first agent payment

Pay any [x402](/agent-payments) service per call in USDC — **no key**, gasless, straight from the step-1 wallet:

```bash
t2 pay https://x402.t2000.ai/t2000/v1/chat/completions \
  --data '{"model":"zai/glm-5.2","messages":[{"role":"user","content":"hi"}]}' \
  --max-price 0.10
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="t2: command not found (after a successful install)">
    The install wrote the binary, but npm's global `bin` directory isn't on your shell's `PATH`. Confirm:

    ```bash
    NPM_BIN="$(npm prefix -g)/bin"
    ls -l "$NPM_BIN"/t2 "$NPM_BIN"/t2000      # do the symlinks exist?
    "$NPM_BIN/t2" --version                    # runs via full path? (package is healthy)
    case ":$PATH:" in *":$NPM_BIN:"*) echo "✓ on PATH";; *) echo "✗ NOT on PATH → $NPM_BIN";; esac
    ```

    If the full-path command prints the version but it's NOT on PATH, add the dir to your shell profile:

    ```bash
    # zsh
    echo "export PATH=\"$(npm prefix -g)/bin:\$PATH\"" >> ~/.zshrc && source ~/.zshrc
    ```

    Then open a **fresh terminal** and re-run `t2 --version`. (`t2000` is a built-in alias for `t2`.)
  </Accordion>
  <Accordion title="EACCES / permission errors on install">
    Don't `sudo npm install` — point npm's global prefix at your home dir instead:

    ```bash
    mkdir -p ~/.npm-global
    npm config set prefix ~/.npm-global
    echo "export PATH=\"$HOME/.npm-global/bin:\$PATH\"" >> ~/.zshrc && source ~/.zshrc
    npm install -g @t2000/cli
    ```

    Or install Node via [nvm](https://github.com/nvm-sh/nvm) / Homebrew, which keep the global prefix user-writable.
  </Accordion>
</AccordionGroup>
