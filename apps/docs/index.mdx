**t2000** is the infra layer for AI agents that move money on Sui. Three packages, one stack:

- **`@t2000/cli`** — the Agent Wallet CLI (`t2`). Gasless USDC + USDsui sends, Cetus swaps, x402 paid API access.
- **`@t2000/mcp`** — the MCP server. Drop into Claude Desktop, Cursor, Windsurf, or any MCP client.
- **`@t2000/sdk`** — the TypeScript SDK. Programmatic wallet + gasless transfers + Cetus routing + x402 pay.

Five-minute setup. No banks, no borders, no fees on the t2000 layer.

## Install

### One-prompt setup (paste into any AI client)

Paste this into Cursor, Claude Code, Claude Desktop, ChatGPT, or any LLM client:

```
Run curl -sL https://t2000.ai/skills/t2000-setup, and use the returned setup instructions to set up my Agent Wallet.
```

The agent fetches the live setup skill — which is **read-only and config-only** (it creates a wallet and wires MCP, never moves funds) — and walks you through it.

<Note>
  **Where the commands actually run.** Agents with access to your local shell — **Cursor**, **Claude Code**, or your terminal — can run the steps with you. Chat clients like **Claude Desktop** and **ChatGPT** run in a cloud sandbox with no access to your machine (and can't reach `t2000.ai` from it), so they'll *read* the skill and hand you the exact commands to paste into your own terminal. Either way the wallet is created locally at `~/.t2000/`, never in the agent's environment — so the [manual install](#manual-install) below is always the reliable path.
</Note>

The skill markdown lives at [`t2000.ai/skills/t2000-setup`](https://t2000.ai/skills/t2000-setup) — `curl` it directly to see what the agent reads.

### Manual install

```bash
npm install -g @t2000/cli
```

Requires Node.js 18+.

```bash
t2 init                              # create a wallet (plain Bech32, 0o600 perms)
t2 receive                           # show address + ANSI QR — fund the wallet
t2 send 5 USDC alice.sui             # gasless USDC send to a SuiNS name
t2 swap 100 USDC SUI                 # best-route swap via Cetus Aggregator
t2 pay https://mpp.t2000.ai/openai/v1/chat/completions --data '…'
t2 mcp install                       # wire Claude Desktop / Cursor / Windsurf
```

### Troubleshooting

<AccordionGroup>
  <Accordion title="t2: command not found (after a successful install)">
    The install wrote the binary, but npm's global `bin` directory isn't on your shell's `PATH`. Confirm:

    ```bash
    npm install -g @t2000/cli

    # Where npm puts global binaries + whether the symlinks exist:
    NPM_BIN="$(npm prefix -g)/bin"
    ls -l "$NPM_BIN"/t2 "$NPM_BIN"/t2000

    # Does it run via full path? (proves the package is healthy)
    "$NPM_BIN/t2" --version          # expect 4.x.x

    # Is that dir actually on PATH?
    case ":$PATH:" in *":$NPM_BIN:"*) echo "✓ on PATH";; *) echo "✗ NOT on PATH → $NPM_BIN";; esac
    ```

    If the full-path command prints the version but the last line says **NOT on PATH**, add the dir to your shell profile:

    ```bash
    # bash
    echo "export PATH=\"$(npm prefix -g)/bin:\$PATH\"" >> ~/.bash_profile && source ~/.bash_profile

    # zsh
    echo "export PATH=\"$(npm prefix -g)/bin:\$PATH\"" >> ~/.zshrc && source ~/.zshrc
    ```

    Then open a **fresh terminal** (shells cache command lookups) and re-run `t2 --version`. As a quick check without editing PATH, `t2000` is a built-in alias for `t2` — both ship in every install.
  </Accordion>
  <Accordion title="EACCES / permission errors on install">
    Your Node was installed somewhere that needs elevated writes (a system `/usr/local` Node). Don't `sudo npm install` — instead point npm's global prefix at your home dir:

    ```bash
    mkdir -p ~/.npm-global
    npm config set prefix ~/.npm-global
    echo "export PATH=\"$HOME/.npm-global/bin:\$PATH\"" >> ~/.zshrc && source ~/.zshrc
    npm install -g @t2000/cli
    ```

    Or install Node via [nvm](https://github.com/nvm-sh/nvm) / Homebrew, which keep the global prefix in a user-writable location.
  </Accordion>
</AccordionGroup>

## What you can build

- **A USDC-paying agent.** Drop `@t2000/cli` into any agent loop; it runs `t2 pay` against 40+ paid APIs without you holding SUI for gas.
- **A wallet-aware AI assistant.** Install `@t2000/mcp` in Claude Desktop or Cursor; the agent gets 9 live wallet tools + 8 skill prompts.
- **A Sui-native fintech.** `@t2000/sdk` gives you gasless USDC + USDsui transfers, Cetus swaps, and x402 pay — all from one class.

## Prerequisites

- A Sui address (the wallet creates one for you).
- ~$1 of USDC to send + a tiny bit of SUI (~0.05) for gas on swaps. USDC + USDsui sends and x402 API payments are **gasless** via Sui's `0x2::balance::send_funds` sponsor.
- An MCP-compatible AI client if you want the agent to call the wallet directly (Claude Desktop / Cursor / Windsurf / Cline / Continue / Codex).

## Why t2000

**Agent Wallet, not general wallet.** Every t2000 surface is narrow on purpose. The CLI sends, swaps, and pays — save, borrow, and rebalance live in [Audric](https://audric.ai). The MCP server exposes 9 tools. The SDK is one class — `T2000`.

**Gasless by default for the things that matter.** USDC and USDsui transfers go through `0x2::balance::send_funds` — the Sui foundation sponsors the gas. Your agent never needs SUI to pay another agent in USDC.

**One stack, one brand.** `@t2000/{sdk,cli,mcp}` are version-locked and ship together. No mix-and-match version drift.
